## How to Use

1. **Before starting a phase**: skim the cheat sheet to preview what you'll learn
2. **During study**: keep the cheat sheet open as a quick reference
3. **After finishing a phase**: close everything and try to reproduce the cheat sheet from memory
4. **Spaced revision**: revisit at 2 weeks, 1 month, 3 months (per [PROGRESS_TRACKER.md](../PROGRESS_TRACKER.md))
5. **Before interviews**: read the cheat sheet + take the self-assessment quiz in the phase file

---

## Format Guide

Each cheat sheet contains:
- **Key Concepts** — definitions you must memorize
- **Important Equations** — math you must be able to write from memory
- **Critical Code Patterns** — snippets you'll write repeatedly in production
- **API Quick Reference** — the 10 most-used library calls
- **Common Mistakes** — traps that catch everyone
- **Interview Quick-Hits** — top 5 questions for this phase with compressed answers
